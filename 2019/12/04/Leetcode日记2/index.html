惟铭的Leetcode日记(#2)

对不起我拖更了…而且还只做了一题(呜呜呜呜)

#2:

You are given two non-empty linked lists representing two non-negative integers. The digits are stored in reverse order and each of their nodes contain a single digit. Add the two numbers and return it as a linked list.

You may assume the two numbers do not contain any leading zero, except the number 0 itself.

Example:

Input: (2 -> 4 -> 3) + (5 -> 6 -> 4)
Output: 7 -> 0 -> 8
Explanation: 342 + 465 = 807

复习

#2 比 #1难度大

题目题目描述提到了链表

那么我们先复习一下链表的知识

https://www.jianshu.com/p/9a0df315090c

https://www.jianshu.com/p/9a4561d42e9a

分析

  • 题目描述为对两个链表元素进行相加之后逆序输出
  • Example 以及是逆序的Input，所以直接加的输出就是逆序的了
  • 两个链表会有长度相同、长度不同、其中一个链表为空这三种情况
  • 链表中，如果返回结果为头结点时，因为是第一个元素，前面已无元素，可初始化一个虚拟头结点
  • 进而虚拟头结点的下一个节点就会是链表真正的头结点
  • 要处理两数相加若超过9，需向前进1，而自身取余的操作(可通过carry=0 或 carry=1 记录进位)

代码

复杂度

时间复杂度：O(max(m,n)) m, n 分别表示l1, l2的长度

空间复杂度：O(max(m,n))

总结

这是看了题解再自己理解完后掌握的一道题

关键在于借助虚拟头结点的操作处理

然后有一点容易被忽略的是：每次计算需要判断carry的值、以免最后一次计算需要补1却没有
