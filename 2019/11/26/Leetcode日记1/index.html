惟铭的Leetcode日记(#1)

前言：

说来惭愧，2019/11/25才开启我的Leetcode之旅。但我为我勇敢跨出的这一步感到开心，毕竟万事开头难，之后更难的便是坚持。本系列我希望两到三天出一篇，一篇也是两三题的量。目前是按照题序，但题序对应的题目并不是从简到难，所以后期可能按照考查内容专题来做…或许压根都坚持不了…hhhh

能改变的很少，但要去做！ (不多BB，开始第一篇日记吧！)

#1:

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

Example:

Given nums = [2, 7, 11, 15], target = 9

Because nums[0] + nums[1] = 2 + 7 = 9

return [0, 1]

分析:

没错，我第一题就不会做了，几乎没头绪

但没关系，我可以慢慢分析，再不济就猜！

再get一遍题干内容：给了一个整数数组，要返回数组中两个数所对应的下标，使得数组中的这两个数之和为目标值target （数组中的元素不能重复使用）

AKA: 就在在数组中找到符合某个规则(或某个等式)的两个数找出来并输出它们的下标

Focus: 我们很容易得到这个等式就是

目标值target = 数组某个数i + 数组中另一个数j

STEP：到这一步，我们问题已经读懂了，分成每个step的话就是。

1：定义数组中第一个值为i=0（数组的序号是从0开始，即第一个值的下标为0)

2:定义数组中i的下一个值为j，即j=i+1

3:通过for循环遍历数组，其中for循环遍历数组一个元素后嵌套多一个for循环遍历当前元素之后的元素，验证是否匹配如上我们总结的等式。若匹配成功，返回这两个数的下标；若在第一个for循环遍历的当前数的基础上的嵌套的新一个for循环遍历当前元素的后续元素无匹配值，便跳出当前值的嵌套for循环，回到第一个for循环开始遍历下一个元素并继续对这个下一个元素进行嵌套for循环...

(我是故意说的这么啰嗦，因为我是刚入门的小白，也是想用最简单的词把思路讲清楚，虽然啰嗦但也通俗，后期慢慢减少赘述直接说方法。人非圣贤孰能无过，若我哪里说的有误或是有更好的讲解方式，不但欢迎指出更会感谢你的质疑~)

接下来是代码实现

这已经是我抓破脑门想出来的办法了

我们来分析一下时间复杂度，因为是for循环再嵌套一个for循环

所以时间复杂度为: O(n^2)

空间复杂度: O(1)

比较低效，程序追求的就是在正确的基础上进行优化

所以，在我查阅题解后发现了一种更好的方法

通过以空间换取速度的方式来优化程序

即采用哈希表

首先补充哈希表的相关知识

看完上面链接的博客的介绍后，我们先看代码再解释

简单来说:

step1: 定义哈希表名为map (此时为空)

step2: 遍历数组，用遍历到的当前元素根据”规则”temp去map中寻找有没符合的值

step3: map存在目标值就返回目标值索引和 i，否则就遍历到的当前元素存入map…

此题哈希表方法的

时间复杂度为: O(n^2)

空间复杂度: O(n)

优化效果的到很大的提升

总结:

艰难的做出第一道题，我第一次做题速度速度很慢，理解过程很痛苦

但我还是拿下了

just keep going !!!
